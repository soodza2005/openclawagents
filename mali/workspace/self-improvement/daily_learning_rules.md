DAILY LEARNING LOG
MALI AI

Purpose

This file records meaningful learning and improvements
that increase Mali's intelligence, reasoning capability,
and usefulness to the Owner.

This is NOT a daily diary.

Only record insights that significantly improve:

- technical understanding
- architecture thinking
- project coordination
- reasoning frameworks
- software engineering capability
- AI team collaboration


--------------------------------------------------

LOGGING RULE

Entries should be created ONLY when a meaningful
improvement in intelligence or engineering capability occurs.

Do NOT write entries for routine work.

Create an entry only if one of the following occurs:

1. A significant technical concept was learned
2. A better reasoning or analysis method was discovered
3. A workflow used repeatedly was optimized
4. A reusable high-value skill was created
5. A previous reasoning mistake revealed a better method
6. AI team coordination improved
7. A new system architecture pattern was identified
8. External research produced a genuinely new insight not previously known
9. An OpenClaw update, security advisory, or trick was discovered and evaluated


--------------------------------------------------

SKILL CREATION THRESHOLD

A new skill should only be created if ALL conditions are met:

- The capability is reusable
- It improves reasoning or engineering performance
- It can be applied in multiple future tasks
- It reduces complexity, errors, or manual effort

Avoid creating skills that are:

- one-time solutions
- minor prompt variations
- duplicated capabilities
- experimental ideas without clear value

EXTERNAL RESEARCH ENTRY

When logging insights gained from external research (web, search, YouTube, articles),
the entry MUST include a RESEARCH TRAIL section.

This is MANDATORY — an entry without a research trail is not valid external research.


SOURCE TYPE

Choose one or more:

- Web Search — Thai
- Web Search — English
- YouTube — Thai
- YouTube — English
- YouTube — Other language
- Official Documentation
- Engineering Article / Blog — Thai
- Engineering Article / Blog — English
- Community Forum — Thai (Pantip, Facebook Group, etc.)
- Community Forum — International (Reddit, Stack Overflow, GitHub)
- Research Paper
- International Source — Japanese
- International Source — Chinese
- International Source — Other


RESEARCH TRAIL

List every search query used and every source visited.
This proves the research actually happened.

Format:

SEARCH QUERIES:
  1. [exact query text]
  2. [exact query text]
  ...

SOURCES VISITED:
  - [YouTube] [video title or channel name] — [country/language if non-English]
  - [Web] [website name or domain] — [brief description]
  - [Docs] [documentation name]
  - [Forum] [Reddit/GitHub/SO thread topic]
  ...

DO record: titles, site names, YouTube channel/video names, query strings
DO record: country or language if source is non-English (e.g., Japanese tech blog, German forum)
DO NOT record: raw URLs (only site name/title is needed)


INSIGHT

[The extracted insight — what was actually learned]


SOURCE QUALITY

Was the source authoritative? (yes/no/partial)
Was the insight verified by multiple sources? (yes/no)
